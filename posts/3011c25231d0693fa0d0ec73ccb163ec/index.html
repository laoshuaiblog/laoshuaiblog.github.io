npm ERR! errno 134

记录一下解决办法：

1、在package.json中添加如下配置，

"dependencies": {
 
        "increase-memory-limit": "^1.0.3",
 
        "cross-env": "^5.0.5"
 
},


2、执行

  npm install increase-memory-limit 或（pnpm install increase-memory-limit）

  npm i cross-env --save-dev 或（ npm i cross-env --save-dev ）

3、 在package.json中添加如下配置，

"scripts": {
    "fix-memory-limit": "cross-env LIMIT=3072  increase-memory-limit"
  },
LIMIT是分配的内存大小，有3027、4096、8192、10240，

4、配置完之后执行

npm run fix-memory-limit

3、执行npm run dev 时如果报错：

'"node --max-old-space-size=3072"' 不是内部或外部命令，也不是可运行的程序
或批处理文件。

解决办法：

在项目中 node_modules.bin 下的 webpack.cmd 和 webpack-dev-server.cmd文件中

去掉双引号, 修改"%_prog%" 为 %_prog%，即可正常运行
